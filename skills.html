Ever since I got my first 800MHz PC in 2002 I've been interested in programming. For years I coded for the Windows Desktop before getting interested in web development in 2005.

In 2008, after my apprenticeship, I became a freelancer for frontend development while learning and messing around with the backend part as well.

Since 2010 I call myself a full-stack web developer. It has been a steady process to find the tools I want to use and learn. And the process does still continue...

Frontend Skills

HTML5 & CSS3

I love playing around with new browser features. Most recently I experimented with the Web MIDI Api.

jQuery

First tool-set I learned when I got into JavaScript, and still on my belt. I also coded jqapi.com.

Backbone.js

First framework I learned when I got into the MVC paradigm. Used it for years and for multiple jobs.

Vue.js

A relatively new MVVM framework I currently use for frontend coding and composing. Love it!

Gulp

I used Grunt for a while before I switched to Gulp for my default frontend build tool.

Browserify

The tool of my choice to structure JavaScript applications. Really dope in combination with Vue.

CoffeeScript

I use this neat language that compiles to JS, to save myself a ton of curly brackets and semicolons.

Sass

The CSS processor I currently use. Plays nice with Bootstrap and Rails. Besides that I also use Stylus.

Bootstrap

HTML/CSS framework I use to prototype websites. Also familiar with Foundation, Semantic-UI and Basscss.

Et cetera

And so much more tools I tried out and will try out. Exciting and fast paced times in frontend tooling.

Backend Skills

Node.js

I've attended the NodeKnockout hackathon 5 years in a row. Came in 3rd in 2012. Fun times!

Ruby

First server-side language I learned before also using Node.js. Attended the RailsRumble hackathon 3 times.

Express

The Node.js web framework of my choice. Also played around with Sails.js and custom made frameworks.

Socket.io

For any realtime communication I use this awesome library. I also use plain Websockets for simple things.

Rails

First framework I learned when I also began to explore the backend. Used it for a couple of years and some jobs.

Sinatra

I use this Rack-based, small, web framework for simple webapps, when Rails would be too much.

Mongoose

The JavaScript adapter I use with Express for my favorite database, the schema-less MongoDB.

Mongoid

And the Ruby adapter I use for MongoDB with Ruby on Rails and Sinatra.

Jekyll

The Ruby tool I use to generate static websites, just like this Blog.

Et cetera

Whatever API's, tools and libraries I got to play with. New things are always fun and exciting.

DevOp Skills

OSX

The OS of my choice for development and everyday tasks. Started with Windows 98, though.

Ubuntu

The oparating system of my choice when it comes to servers. I deploy all my webapps to this environment.

Chrome

My default browser for webdev and everyday lurking around. Also use Firefox for double-checking.

Atom

After using Sublime Text for years, I switched to this open-source editor and never looked back.

Git

The version control tool of my choice. For larger projects I mostly use it with Git Flow.

Nginx

The webserver I trust for handling all my HTTP requests. I am familiar and used Apache2, too.

Docker

The latest tool I play around with for deploying and hosting Node.js or Ruby apps. Also for testing Linux skills.

Et cetera

Just like in the front- and backend, I think it's important to try out new tools and find the right one for the job.
